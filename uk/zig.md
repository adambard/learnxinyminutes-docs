[Zig][ziglang] — мова програмування, що прагне замінити C.

**УВАГА**: цей документ передбачає, що ви розумієте базові концепції в інформатиці, такі як вказівники, стекова та купова пам'ять тощо. Рекомендується попереднє знання мови C.

## Короткий огляд: Zig порівняно з C

- Синтаксис здебільшого такий самий, але з покращеннями (менше неоднозначності).
- Zig вводить простори імен.
- Механізм `try` і `catch`, який є зручним, ефективним і необов'язковим.
- Більшість невизначених поведінок (UB) з C виправлено.
- Порівняно з C, роботизовані вказівники безпечніші й рідше потрібні.
  - Система типів розрізняє вказівник на одне значення, на кілька значень тощо.
  - Перевагу надають зрізам (slices) — структурам із вказівником і розміром, відомим під час виконання, що відповідає більшості реальних випадків використання вказівників.
- Деякі довільні обмеження мови C знято. Наприклад, переліки, структури та об'єднання можуть мати функції.
- Простий доступ до операцій SIMD (базова математика над векторами).
- Zig надає як низькорівневі можливості C, так і ті, що зазвичай реалізуються через розширення компілятора.
  Наприклад: упаковані структури (packed structs).
- Розширена стандартна бібліотека, що включає структури даних та алгоритми.
- Крос-компіляція доступна за замовчуванням, без жодних залежностей.
  Надаються різні реалізації libc для спрощення процесу.
  Крос-компіляція працює з будь-якої операційної системи та архітектури і для будь-якої з них.

## Мова Zig

```zig
//! Документація верхнього рівня.

// Звичайний коментар.
```

### Hello world.

```zig
// Імпортуємо стандартну бібліотеку, доступну через константу "std".
const std = @import("std");

// "info" тепер посилається на функцію "std.log.info".
const info = std.log.info;

// Класичний hello world.
// синтаксис: [pub] fn <ім'я-функції>(<аргументи>) <тип-що-повертається> { <тіло> }
pub fn main() void {
    // На відміну від функцій C, функції Zig мають фіксовану кількість аргументів.
    // У C: "printf" приймає будь-яку кількість аргументів.
    // У Zig: std.log.info приймає формат і список елементів для виведення.
    info("привіт, світе", .{});  // .{} — порожній анонімний кортеж.
}
```

### Булеві значення, цілі числа та числа з рухомою комою.

```zig
// Булеві значення.
// Для булевих операцій перевагу надають ключовим словам, а не операторам.
print("{}\n{}\n{}\n", .{
    true and false,
    true or false,
    !true,
});

// Цілі числа.
const one_plus_one: i32 = 1 + 1;
print("1 + 1 = {}\n", .{one_plus_one}); // 2

// Числа з рухомою комою.
const seven_div_three: f32 = 7.0 / 3.0;
print("7.0 / 3.0 = {}\n", .{seven_div_three}); // 2.33333325e+00

// Цілі числа мають довільну розрядність.
var myvar: u10 = 5; // беззнакове 10-бітне ціле число
// Корисно, наприклад, для читання мережевих пакетів або складних бінарних форматів.

// Представлення чисел значно покращено порівняно з C.
const one_billion = 1_000_000_000;         // Десяткове.
const binary_mask = 0b1_1111_1111;         // Двійкове. Прикл.: мережева маска.
const permissions = 0o7_5_5;               // Вісімкове. Прикл.: права Unix.
const big_address = 0xFF80_0000_0000_0000; // Шістнадцяткове. Прикл.: адреса IPv6.


// Оператори переповнення: повідомляють компілятору, коли переповнення допустиме.
var i: u8 = 0;  // "i" — беззнакове 8-бітне ціле число
i  -= 1;        // помилка переповнення під час виконання (беззнакове значення завжди додатне)
i -%= 1;        // допустимо (оператор з огортанням), i == 255

// Оператори насичення: значення "прилипають" до нижньої та верхньої меж.
var i: u8 = 200;   // "i" — беззнакове 8-бітне ціле число (значення від 0 до 255)
i  +| 100 == 255   // u8: не може бути більше за 255
i  -| 300 == 0     // беззнакове, не може бути нижче за 0
i  *| 2   == 255   // u8: не може бути більше за 255
i <<| 8   == 255   // u8: не може бути більше за 255
```

### Масиви.

```zig
// Масив — чітко визначена структура з атрибутом довжини (len).

// 5-байтовий масив із невизначеним вмістом (сміття на стеку).
var array1: [5]u8 = undefined;

// 5-байтовий масив із визначеним вмістом.
var array2 = [_]u8{ 1, 2, 3, 4, 5 };
// [_] означає, що компілятор знає довжину під час компіляції.

// 1000-байтовий масив із визначеним вмістом (0).
var array3 = [_]u8{0} ** 1000;

// Ще один 1000-байтовий масив із визначеним вмістом.
// Вміст надається функцією "foo", що викликається під час компіляції та
// дозволяє виконувати складну ініціалізацію.
var array4 = [_]u8{foo()} ** 1000;

// У будь-якому випадку, array.len повертає довжину масиву:
// array1.len і array2.len дорівнюють 5, array3.len і array4.len дорівнюють 1000.


// Зміна та доступ до вмісту масивів.

// Масив із 10 невизначених 32-бітних цілих чисел зі знаком.
var some_integers: [10]i32 = undefined;

some_integers[0] = 30; // перший елемент масиву тепер дорівнює 30

var x = some_integers[0]; // "x" тепер дорівнює 30, тип виводиться автоматично.
var y = some_integers[1]; // Другий елемент масиву не визначений.
                          // "y" отримає значення сміття зі стеку (без помилки під час виконання).

// Масив із 10 невизначених 32-бітних цілих чисел зі знаком.
var some_integers: [10]i32 = undefined;

var z = some_integers[20]; // індекс > розмір масиву, помилка компіляції.

// Під час виконання ми проходимо елементи "some_integers" за індексом.
// Індекс i = 20, тоді ми намагаємося:
try some_integers[i]; // Помилка під час виконання 'index out of bounds' (вихід за межі).
                      // Ключове слово "try" необхідне при зверненні до масиву за індексом,
                      // оскільки існує потенційна помилка під час виконання.
                      // Докладніше про це пізніше.
```

### Рядки.

```zig
// Проста рядкова константа.
const greetings = "привіт";
// ... що еквівалентно:
const greetings: *const [6:0]u8 = "привіт";
// Іншими словами: "greetings" — константне значення, вказівник на константний масив із 6
// елементів (8-бітних беззнакових цілих чисел) із зайвим '0' у кінці.
// Зайвий "0" називається "сторожовим значенням" (sentinel value).

print("рядок: {s}\n", .{greetings});

// Це досить точно відображає рядки C. Проте рядки Zig — структури,
// не потрібна функція "strlen" для обчислення їхнього розміру.
// greetings.len == 6
```

### Зрізи.

```zig
// Зріз — це вказівник і розмір, масив без відомого під час компіляції розміру.
// Зрізи мають перевірки виходу за межі під час виконання.

const array = [_]u8{1,2,3,4,5};     // [_] = масив із розміром, відомим під час компіляції.
```

### Вказівники.

```zig
// Вказівник на значення можна створити за допомогою "&".
const x: i32 = 1;
const pointer: *i32 = &x;  // "pointer" — вказівник на змінну i32 "x".
print("1 = {}, {}\n", .{x, pointer});

// До значень вказівника звертаються та змінюють їх через ".*".
if (pointer.* == 1) {
    print("значення x == {}\n", .{pointer.*});
}

// ".?" — скорочення для "orelse unreachable".
const foo = pointer.?; // Отримати вказане значення, інакше аварійне завершення.
```

### Необов'язкові значення (?\<тип\>).

```zig
// Необов'язкове значення може бути будь-якого типу або null.

// Приклад: "optional_value" може бути або "null", або беззнаковим 32-бітним цілим числом.
var optional_value: ?u32 = null; // optional_value == null
optional_value = 42;             // optional_value != null

// "some_function" повертає ?u32
var x = some_function();
if (x) |value| {
    // Якщо "some_function" повернула значення.
    // Щось робимо з 'value'.
}
```

### Помилки.

```zig
// Zig надає уніфікований спосіб вираження помилок.

// Помилки визначаються у переліках помилок, наприклад:
const Error = error {
    ПереглядБудьЯкогоСеріалуNetflix,
    СидітиВTwitter,
};

// Звичайні переліки виражаються так само, але з ключовим словом "enum".
const SuccessStory = enum {
    ЗайматисяСпортом,
    ПрочитатиКнигу,
};


// Об'єднання з помилкою (!).
// "mylife" є або помилкою, або звичайним значенням.
var mylife: Error!SuccessStory = Error.СидітиВTwitter;
// mylife — помилка. Сумно.

mylife = SuccessStory.ПрочитатиКнигу;
// Тепер mylife — це перелік.


// Zig постачається з багатьма наперед визначеними помилками. Наприклад:
const value: anyerror!u32 = error.Broken;


// Обробка помилок.

// Кілька прикладів помилок.
const Error = error {
    Неочікувана,
    Автентифікація,
};

// "some_function" може повернути або "Error", або ціле число.
fn some_function() Error!u8 {
    return Error.Неочікувана; // Вона повертає помилку.
}

// Помилки можна "перехопити" (catch) без проміжної змінної.
var value = some_function() catch |err| switch(err) {
    Error.Неочікувана    => return err,   // Повертає помилку.
    Error.Автентифікація => unreachable,  // Не очікується. Аварійне завершення програми.
    else                 => unreachable,
};

// Помилку можна "перехопити" не надаючи їй імені.
const unwrapped = some_function() catch 1234; // "unwrapped" = 1234

// "try" — дуже зручне скорочення для "catch |err| return err".
var value = try some_function();
// Якщо "some_function" зазнає невдачі, поточна функція зупиняється і повертає помилку.
// "value" може містити лише коректне значення — помилка вже оброблена через "try".
```

### Керування потоком виконання.

```zig
// Умовне розгалуження.

if (condition) {
    ...
}
else {
    ...
}

// Тернарний оператор.
var value = if (condition) x else y;

// Скорочення для "if (x) x else 0"
var value = x orelse 0;

// Якщо "a" — необов'язкове значення, що може містити значення.
if (a) |value| {
    print("значення: {}\n", .{value});
}
else {
    print("'a' дорівнює null\n", .{});
}

// Отримати вказівник на значення (якщо воно існує).
if (a) |*value| { value.* += 1; }


// Цикли.

// Приклади синтаксису:
//   while (умова) інструкція
//   while (умова) : (інструкція-наприкінці-ітерації) інструкція
//
//   for (ітерований-об'єкт) інструкція
//   for (ітерований-об'єкт) |захоплення| інструкція
//   for (ітерований-об'єкт, ітерований-об'єкт...) |захоплення, захоплення...| інструкція
//   for (ітерований-об'єкт) інструкція else інструкція

// Примітка: цикли однаково працюють як з масивами, так і зі зрізами.

// Простий цикл "while".
while (i < 10) { i += 1; }

// Цикл while з "виразом продовження"
// (виразом, що виконується як останній вираз циклу).
while (i < 10) : (i += 1) { ... }
// Те саме, з більш складним виразом продовження (блок коду).
while (i * j < 2000) : ({ i *= 2; j *= 3; }) { ... }

// Простий цикл for по діапазону.
for (0..10) |i| { sum += i; }

// Щоб ітерувати по частині зрізу, зробіть повторний зріз.
for (items[0..1]) |value| { sum += value; }

// Ітерація по кожному елементу масиву (або зрізу).
for (items) |value| { sum += value; }

// Ітерувати та отримувати вказівники на значення замість копій.
for (&items) |*value| { value.* += 1; }

// Можна ітерувати по кількох діапазонах одночасно.
for (0..10, 10..20) |i, j| { sum += i * j; }

// Ітерація з індексом.
for (items, 0..) |value, i| { print("val[{}] = {}\n", .{i, value}); }

// Ітерація з вказівником та індексом.
for (&items, 0..) |*value, i| { print("val[{}] = {}\n", .{i, value}); value.* += 1; }


// Break і continue підтримуються.
for (items) |value| {
    if (value == 0)  { continue; }
    if (value >= 10) { break;    }
    // ...
}

// Цикли for також можна використовувати як вирази.
// Подібно до циклів while, коли ви виходите з циклу for через break,
// гілка else не виконується.
var sum: i32 = 0;
// Цикл "for" має надати значення,
// яким у цьому випадку буде значення "else".
const result = for (items) |value| {
    // Додати "value" до "sum", якщо воно не "null", інакше додати 0
    sum += value orelse 0;
} else 0; // result == 0


// Щоб встановити "result" у значення "sum",
// вийдіть із нього через break на останній ітерації.
const result = for (items, 1..) |value, i| {
    sum += value orelse 0;

    // Перевіряємо, чи це остання ітерація
    if (i == items.len) {
        // Виходимо зі значенням через break
        break sum; // result == sum
    }
} else unreachable;
```

### Мітки.

```zig
// Мітки — це спосіб назвати інструкцію або місце в коді.
// Мітки можна використовувати для "continue" або "break" у вкладеному циклі.
outer: for ([_]i32{ 1, 2, 3, 4, 5, 6, 7, 8 }) |_| {
    for ([_]i32{ 1, 2, 3, 4, 5 }) |_| {
        count += 1;
        continue :outer; // "continue" для першого циклу.
    }
} // count = 8
outer: for ([_]i32{ 1, 2, 3, 4, 5, 6, 7, 8 }) |_| {
    for ([_]i32{ 1, 2, 3, 4, 5 }) |_| {
        count += 1;
        break :outer; // "break" для першого циклу.
    }
} // count = 1


// Мітки також можна використовувати для повернення значення з блоку.
var y: i32 = 5;
const x = blk: {
    y += 1;
    break :blk y; // Тепер "x" дорівнює 6.
};
// Корисно, наприклад, у виразах "for else" (описано нижче).

// Цикли for можна використовувати як вирази.
// Коли ви виходите з циклу for через break, гілка else не виконується.
// УВАГА: неочевидна поведінка.
//      Цикл "for" виконається, потім виконається блок "else".
//      Після ключового слова "else" має стояти значення, що буде надано "result".
//      Нижче показана інша форма.
var sum: u8 = 0;
const result = for (items) |value| {
    sum += value;
} else 8; // result = 8

// У цьому випадку після "else" також стоїть значення.
// Проте синтаксис інший: він має мітку.
// Замість значення — мітка, за якою йде блок коду, що дозволяє
// виконати додаткові дії перед поверненням значення (дивіться виклик "break").
const result = for (items) |value| { // Спочатку: цикл.
    sum += value;
} else blk: {                        // Потім: блок "else".
    std.log.info("виконується ПІСЛЯ циклу!", .{});
    break :blk sum; // Значення "sum" замінить мітку "blk".
};
```

### Switch.

```zig
// Аналог switch з C, але дещо розширений.
// Синтаксис:
//   switch (значення) {
//       шаблон => вираз,
//       шаблон => вираз,
//       else    => вираз
//   };

// Switch, що перевіряє лише прості значення.
var x = switch(value) {
    Error.Неочікувана    => return err,
    Error.Автентифікація => unreachable,
    else                 => unreachable,
};

// Дещо складніший switch, що приймає діапазон значень:
const foo: i32 = 0;
const bar = switch (foo) {
    0                        => "нуль",
    1...std.math.maxInt(i32) => "додатне",
    else                     => "від'ємне",
};
```

### Структури.

```zig
// Структура, що містить одне значення.
const Full = struct {
    number: u16,
};

// Упакована структура з гарантованим розташуванням у пам'яті.
const Divided = packed struct {
    half1: u8,
    quarter3: u4,
    quarter4: u4,
};

// Point — константа, що представляє структуру з двох u32: "x" та "y".
// "x" має значення за замовчуванням, що було неможливим у C.
const Point = struct {
    x: u32 = 1, // значення за замовчуванням
    y: u32,
};

// Змінна "p" — новий Point із x = 1 (значення за замовчуванням) і y = 2.
var p = Point{ .y = 2 };

// Поля доступні звичайним чином через крапку: змінна.поле.
print("p.x: {}\n", .{p.x}); // 1
print("p.y: {}\n", .{p.y}); // 2


// Структура також може містити публічні константи та функції.
const Point = struct {
    pub const some_constant = 30;

    x: u32,
    y: u32,

    // Ця функція "init" створює Point і повертає його.
    pub fn init() Point {
        return Point{ .x = 0, .y = 0 };
    }
};


// Як звертатися до публічної константи структури.
// Значення доступне не через "екземпляр" структури, а через
// константу, що представляє визначення структури (Point).
print("константа: {}\n", .{Point.some_constant});

// Наявність функції "init" є доволі ідіоматичним у стандартній бібліотеці.
// Докладніше про це пізніше.
var p = Point.init();
print("p.x: {}\n", .{p.x}); // p.x = 0
print("p.y: {}\n", .{p.y}); // p.y = 0


// Структури часто мають функції для зміни свого стану, подібно до
// об'єктно-орієнтованого програмування.
const Point = struct {
    const Self = @This(); // Посилається на власний тип (пізніше — "Point").

    x: u32,
    y: u32,

    // Зверніть увагу на сигнатуру. Перший аргумент має тип *Self: "self" —
    // це вказівник на екземпляр структури.
    // Це дозволяє використовувати ту саму "крапкову" нотацію, що й в ООП,
    // наприклад "instance.set(x,y)".
    // Дивіться наступний приклад.
    pub fn set(self: *Self, x: u32, y: u32) void {
        self.x = x;
        self.y = y;
    }

    // Знову зверніть увагу на сигнатуру. Перший аргумент має тип Self (не *Self),
    // це не вказівник. У цьому випадку "self" посилається на екземпляр
    // структури, але не може бути змінений.
    pub fn getx(self: Self) u32 {
        return self.x;
    }

    // PS: дві попередні функції можуть бути дещо зайвими.
    //     Атрибути можна змінювати безпосередньо, без функцій-аксесорів.
    //     Це лише приклад.
};

// Використаємо попередню структуру.
var p = Point{ .x = 0, .y = 0 }; // змінна "p" є Point.

p.set(10, 30); // атрибути x і y змінної "p" змінюються через функцію "set".
print("p.x: {}\n", .{p.x}); // 10
print("p.y: {}\n", .{p.y}); // 30

// У C:
//   1. Ми б написали щось на кшталт: point_set(p, 10, 30).
//   2. Оскільки всі функції знаходяться в одному просторі імен, було б
//      дуже незручно створювати функції з різними іменами для різних
//      структур. Безліч довгих імен, важко читати.
//
// У Zig структури надають простори імен для своїх функцій.
// Різні структури можуть мати функції з однаковими іменами,
// що покращує читабельність.
```

### Мічені об'єднання (tagged unions).

```zig
// Об'єднання можна оголошувати з типом-міткою у вигляді переліку, що дозволяє
// використовувати їх у виразах switch.

const MaybeEnum = enum {
    success,
    failure,
};

const Maybe = union(MaybeEnum) {
    success: u8,
    failure: []const u8,
};

// Перше значення: успіх!
const yay = Maybe{ .success = 42 };
switch (yay) {
    .success => |value|     std.log.info("успіх: {}", .{value}),
    .failure => |err_msg|   std.log.info("невдача: {}", .{err_msg}),
}

// Друге значення: невдача! :(
const nay = Maybe{ .failure = "Я був надто лінивий" };
switch (nay) {
    .success => |value|     std.log.info("успіх: {}", .{value}),
    .failure => |err_msg|   std.log.info("невдача: {}", .{err_msg}),
}
```

### Defer та errdefer.

```zig
// Гарантувати виконання дії (одиночної інструкції або блоку коду)
// перед завершенням поточної області видимості (функції, блоку коду).
// Навіть у разі помилки ця дія буде виконана.
// Корисно для виділення пам'яті та керування ресурсами загалом.

pub fn main() void {
    // Має виконатися наприкінці функції.
    defer print("третій!\n", .{});

    {
        // Останній елемент своєї області видимості: виконається негайно.
        defer print("перший!\n", .{});
    }

    print("другий!\n", .{});
}

fn hello_world() void {
    defer print("кінець функції\n", .{}); // після "привіт, світе!"

    print("привіт, світе!\n", .{});
}

// errdefer виконує інструкцію (або блок коду) лише у разі помилки.
fn second_hello_world() !void {
    errdefer print("2. щось пішло не так!\n", .{}); // якщо "foo" зазнає невдачі.
    defer    print("1. second hello world\n", .{});  // виконується після "foo"

    try foo();
}
// Оператори defer можна уявити як стек: перший доданий виконується останнім.
```

### Алокатори пам'яті.
Пам'ять не керується безпосередньо у стандартній бібліотеці; замість цього щоразу, коли потрібна операція з пам'яттю, використовується "алокатор". Таким чином стандартна бібліотека дозволяє розробникам керувати пам'яттю так, як їм потрібно, через структури, що називаються "алокаторами" і обробляють усі операції з пам'яттю.

**ПРИМІТКА**: вибір алокатора виходить за межі цього документа.
Про це можна написати цілу книгу.
Проте ось кілька прикладів, щоб дати уявлення про те, що вас чекає:

- `page_allocator`.
  Виділяє цілу сторінку пам'яті щоразу, коли запитується пам'ять.
  Дуже простий, дуже примітивний, дуже марнотратний.
- `GeneralPurposeAllocator`.
  Спочатку отримує певний обсяг пам'яті та керує кількома "відрами" пам'яті,
  щоб скоротити кількість виділень.
  Дещо складний. Можна поєднувати з іншими алокаторами.
  Може виявляти витоки та надавати корисну інформацію для їх пошуку.
- `FixedBufferAllocator`.
  Використовує фіксований буфер як джерело пам'яті, не звертається до ядра.
  Дуже простий, обмежений і марнотратний (не може звільняти пам'ять), але дуже швидкий.
- `ArenaAllocator`.
  Дозволяє звільнити всю виділену пам'ять одним викликом.
  Використовується разом з іншим алокатором.
  Дуже простий спосіб уникнути витоків.

Перший приклад.

```zig
// "!void" означає, що функція не повертає жодного значення, крім помилок.
// У цьому випадку ми намагаємося виділити пам'ять, що може зазнати невдачі.
fn foo() !void {
    // У цьому прикладі використовуємо page_allocator.
    var allocator = std.heap.page_allocator;

    // "list" — ArrayList 8-бітних беззнакових цілих чисел.
    // ArrayList — це суцільний, зростаючий список елементів у пам'яті.
    var list = try ArrayList(u8).initAllocated(allocator);
    defer list.deinit(); // Звільнити пам'ять наприкінці області видимості. Витік неможливий.
    // "defer" дозволяє виразити звільнення пам'яті одразу після її виділення,
    // незалежно від складності функції (цикли, умови тощо).

    list.add(5); // Тут виділяється пам'ять за допомогою наданого алокатора.

    for (list.items) |item| {
        std.debug.print("елемент: {}\n", .{item});
    }
}
```

### Виділення пам'яті у поєднанні з обробкою помилок і defer.

```zig
fn some_memory_allocation_example() !void {
    // Виділення пам'яті може зазнати невдачі, тому ми "try"-маємо виділити пам'ять і
    // у разі помилки поточна функція повертає її.
    var buf = try page_allocator.alloc(u8, 10);
    // Відкладаємо звільнення пам'яті одразу після виділення.
    // Відбудеться навіть у разі помилки.
    defer page_allocator.free(buf);

    // Друге виділення.
    // У разі невдачі перше виділення коректно звільняється.
    var buf2 = try page_allocator.alloc(u8, 10);
    defer page_allocator.free(buf2);

    // У разі невдачі обидва попередні виділення коректно звільняються.
    try foo();
    try bar();

    // ...
}
```

### Алокатори пам'яті: знайомство зі стандартною бібліотекою.

```zig
// Алокатори: 4 основні функції, які треба знати
//   одне_значення = create (тип)
//   destroy (одне_значення)
//   зріз = alloc (тип, розмір)
//   free (зріз)

// Page Allocator
fn page_allocator_fn() !void {
    var slice = try std.heap.page_allocator.alloc(u8, 3);
    defer std.heap.page_allocator.free(slice);

    // playing_with_a_slice(slice);
}

// GeneralPurposeAllocator
fn general_purpose_allocator_fn() !void {
    // GeneralPurposeAllocator потребує налаштування.
    // У цьому випадку ми хочемо відстежувати витоки пам'яті.
    const config = .{.safety = true};
    var gpa = std.heap.GeneralPurposeAllocator(config){};
    defer _ = gpa.deinit();

    const allocator = gpa.allocator();

    var slice = try allocator.alloc(u8, 3);
    defer allocator.free(slice);

    // playing_with_a_slice(slice);
}

// FixedBufferAllocator
fn fixed_buffer_allocator_fn() !void {
    var buffer = [_]u8{0} ** 1000; // масив із 1000 u8, ініціалізованих нулем.
    var fba  = std.heap.FixedBufferAllocator.init(buffer[0..]);
    // Примітка: buffer[0..] — це спосіб створити зріз з масиву.
    //            Оскільки функція приймає зріз, а не масив,
    //            це задовольняє систему типів.

    var allocator = fba.allocator();

    var slice = try allocator.alloc(u8, 3);
    // "free" не потрібен, пам'ять не можна звільнити з FixedBufferAllocator.
    // defer allocator.free(slice);

    // playing_with_a_slice(slice);
}

// ArenaAllocator
fn arena_allocator_fn() !void {
    // Нагадування: arena не виділяє пам'ять сам, він використовує внутрішній алокатор.
    // У цьому випадку поєднуємо arena allocator із page allocator.
    var arena = std.heap.arena_allocator.init(std.heap.page_allocator);
    defer arena.deinit(); // кінець функції = всі виділення звільнено.

    var allocator = arena.allocator();

    const slice = try allocator.alloc(u8, 3);
    // "free" не потрібен, пам'ять так чи інакше буде звільнена.

    // playing_with_a_slice(slice);
}


// Поєднання алокаторів general purpose та arena. Обидва дуже корисні,
// і їх поєднання варто мати в арсеналі кожного розробника.
fn gpa_arena_allocator_fn() !void {
    const config = .{.safety = true};
    var gpa = std.heap.GeneralPurposeAllocator(config){};
    defer _ = gpa.deinit();

    const gpa_allocator = gpa.allocator();

    var arena = arena_allocator.init(gpa_allocator);
    defer arena.deinit();

    const allocator = arena.allocator();

    var slice = try allocator.alloc(u8, 3);
    defer allocator.free(slice);

    // playing_with_a_slice(slice);
}
```

### Comptime (обчислення під час компіляції).

```zig
// Comptime — це спосіб уникнути препроцесора.
// Ідея проста: виконувати код під час компіляції.

inline fn max(comptime T: type, a: T, b: T) T {
    return if (a > b) a else b;
}

var res = max(u64, 1, 2);
var res = max(f32, 10.50, 32.19);


// Comptime: створення узагальнених структур.

fn List(comptime T: type) type {
    return struct {
        items: []T,

        fn init()   ... { ... }
        fn deinit() ... { ... }
        fn do()     ... { ... }
    };
}

const MyList = List(u8);


// використання
var list = MyList{
    .items = ... // виділення пам'яті
};

list.items[0] = 10;
```

### Вбудовані функції компілятора.

Компілятор має спеціальні функції, що називаються "вбудованими" (built-ins) і починаються з "@".
Їх більше сотні, і вони дозволяють виконувати дуже низькорівневі операції:

- помилки, логування та перевірки під час компіляції
- приведення та перетворення типів, у тому числі небезпечне
- керування вирівнюванням
- маніпуляції з пам'яттю (наприклад, отримання зсуву байту поля у структурі)
- виклик функцій під час компіляції
- включення заголовків C для прозорого виклику функцій C
- атомарні операції
- вбудовування файлів у виконуваний файл (@embedFile)
- маніпуляції зі стековими фреймами (наприклад, для асинхронних функцій)
- тощо.

Приклад: переліки не є цілими числами, їх потрібно перетворювати за допомогою вбудованої функції.

```zig
const Value = enum { zero, stuff, blah };
if (@enumToInt(Value.zero)  == 0) { ... }
if (@enumToInt(Value.stuff) == 1) { ... }
if (@enumToInt(Value.blah)  == 2) { ... }
```

### Кілька заходів захисту від помилок у мові Zig.

- Простори імен: конфлікти імен легко уникнути.
  На практиці це означає уніфікований API між різними структурами (типами даних).
- Переліки не є цілими числами. Порівняння переліку з цілим числом потребує перетворення.
- Явні приведення типів; неявне приведення існує, але обмежене.
  Типи трохи жорсткіші, ніж у C, ось кілька прикладів:
    Вказівники не є цілими числами, потрібне явне перетворення.
    Не можна втратити точність випадково; неявне приведення дозволено лише там, де точність не може бути втрачена.
    Об'єднання не можна перетлумачити випадково (в об'єднанні з цілим числом і числом з рухомою комою не можна сплутати одне з одним).
    Тощо.
- Усунення більшості невизначених поведінок C (UB); коли компілятор натрапляє на таку, він зупиняється.
- Структури зрізів і масивів переважають над вказівниками.
  Типи, що примусово перевіряються компілятором, менш схильні до помилок, ніж маніпуляції з вказівниками.
- Числове переповнення призводить до помилки, якщо явно не дозволено за допомогою операторів огортання.
- Механізм `try` і `catch`.
  Він зручний, тривіально реалізований (простий перелік помилок) і займає майже жодного місця та часу обчислення.
- Невикористані змінні вважаються помилкою компілятора.
- Існує багато типів вказівників для представлення того, на що вказується.
  Наприклад: це одне значення чи масив, чи відома довжина тощо.
- Атрибути структур потребують значення, і все ще можна задати невизначене значення (сміття зі стеку), але принаймні це явно позначено як `undefined`.
